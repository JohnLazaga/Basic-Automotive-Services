Setting up a branch — the easy way

A step-by-step guide for the Pilot branch. No technical experience needed — just follow each step in order. Takes about 30–45 minutes.

What this does (in plain words)

You are turning a small, always-on computer at the shop (we'll call it the mini-PC) into the shop's own private copy of the Basic by JMSI app. Once it's set up:

  • Staff open the app from their own phones or computers using the shop's Wi-Fi — nothing to install on their devices.
  • It keeps working even if the internet goes down, because everything lives on the mini-PC.
  • The parts price list comes from your existing SQL Server computer.
  • You (the owner) stay in control — staff can't change the app itself.

Before you start — what you need

  • A mini-PC (a small Windows computer that can stay switched on all day) at the shop.
  • A keyboard, mouse and monitor connected to it — just for this setup.
  • The mini-PC connected to the shop's internet / Wi-Fi.
  • The name or address of the computer that runs SQL Server (your parts list). Ask whoever manages that PC if you're not sure. You can also do this part later.
  • About 30–45 minutes.
Do everything below on the mini-PC itself (with its keyboard and monitor), unless a step says otherwise.

The steps

1 Install "Node" (the engine that runs the app)

  1. On the mini-PC, open a web browser (Edge or Chrome) and go to nodejs.org.
  2. Click the big green button that says "LTS" (the recommended version). A file downloads.
  3. Open the downloaded file. Click Next → Next → (accept) → Install. If Windows asks "Do you want to allow this app to make changes?", click Yes.
  4. When it finishes, click Finish.
You don't need to see anything yet — Node works quietly in the background.

2 Install "Git" (used to download and update the app)

  1. Go to git-scm.com/download/win. The download starts automatically.
  2. Open the downloaded file. Keep clicking Next through every screen (the default choices are fine), then Install, then Finish. Click Yes if Windows asks permission.

3 Download the app onto the mini-PC

  1. Press the Windows key, type cmd, and press Enter. A black window opens — this is the Command Prompt. Don't worry, you only type two lines.
  2. Type this line and press Enter:
Type this exactly, then Entercd C:\

Then type this line and press Enter:

Type this exactly, then Entergit clone https://github.com/JohnLazaga/Basic-Automotive-Services.git basic
What you should see: lines of text scrolling, ending with something like "done." A new folder C:\basic now exists with the app inside. You can close the black window.

4 Create the branch's settings file

This tiny file tells the mini-PC that it is the Pilot branch.

  1. Press Windows, type Notepad, press Enter.
  2. Copy the five lines below into Notepad exactly:
Copy these 5 lines into Notepad@echo off set "SLUG=pilot" set "BRANCH=Pilot" set "PORT=8790" set "APP_DIR=..\dist\pilot"
  1. In Notepad click File → Save As.
  2. Navigate to the folder C:\basic\branch-server.
  3. In the "Save as type" box at the bottom, choose All Files (*.*). This is important.
  4. In "File name" type exactly: branch.config.cmd
  5. Click Save. Close Notepad.
Common mistake: if you skip the "All Files" step, Windows secretly names the file branch.config.cmd.txt and it won't work. If unsure, open the C:\basic\branch-server folder and check the file is named exactly branch.config.cmd (no .txt on the end).

5 Start the app

  1. Open the folder C:\basic\branch-server (use File Explorer).
  2. Find the file called update.cmd and double-click it.
  3. A black window opens and does three things automatically: gets the latest app, builds the Pilot version, and starts it. Leave this window open — it's the app running.
What you should see: a line in the window mentioning admin and admin — that's your first login. And a line like listening on ... 8790.

6 Open the app and sign in

  1. On the mini-PC, open a browser and go to: http://localhost:8790/
  2. You'll see the Basic by JMSI sign-in screen. Sign in with username admin and password admin.
Do this immediately: go to Settings → change the admin password to something only you know. Then open Accounts & Roles and add each staff member their own login (name, username, password, role). Keep the admin account for yourself only.

7 Give the mini-PC a fixed address (so staff phones can find it)

Right now only the mini-PC can open the app. To let staff phones/computers open it, the mini-PC needs a fixed address on the shop network.

  1. The easiest way: ask your internet installer / whoever set up the router to "reserve a static IP" for this computer — it's a 2-minute job for them.
  2. Once you have that address (it looks like 192.168.1.50), staff open the app on their phones at: http://THAT-ADDRESS:8790/ (on the shop Wi-Fi).
Good to know: whatever address a device uses to open the app, everything just works — you don't set anything up on the phones. Bookmark the address on each phone's home screen.

8 Connect the parts price list (SQL Server)

This pulls your parts and prices from your existing SQL Server computer. If you're not ready, skip this for now — the app still works, you'll just type parts manually until it's connected.

  1. In the app: Settings → Parts catalog — SQL Server.
  2. Fill in the boxes (mostly pre-filled for you):
BoxWhat to put
SQL Server instanceThe parts computer's name, then \MSSQLSERVER01 — for example SHOP-SERVER\MSSQLSERVER01. (Ask us for the exact value.)
DatabasejasRegaladoDB (already filled in)
AuthenticationChoose SQL login and enter the username/password if the parts PC is a different computer. (Choose Windows only if told to.)
  1. Click Test connection. You should see a count of parts and a few samples.
  2. Click Attach & sync. Done — parts now auto-fill when staff type a code.
If Test fails: the parts computer usually needs a one-time setup (turning on network access + a firewall opening). Send us the error message and we'll walk you through it — it's quick.

9 Make the app start by itself when the mini-PC turns on

  1. Open the folder C:\basic\branch-server.
  2. Right-click the file start.cmd → Show more options → Create shortcut. A shortcut appears.
  3. Press Windows + R, type shell:startup, press Enter. A folder opens.
  4. Drag the shortcut you made into that folder.
Now: every time the mini-PC restarts, the app starts on its own. (You can close the black window and it stays running from now on.)

Using it every day

  • Staff: open the bookmarked address on their phone (on shop Wi-Fi) and sign in with their own login.
  • The mini-PC: just leave it switched on. That's it.

Updating the app (when we send an improvement)

On the mini-PC, open C:\basic\branch-server and double-click update.cmd. It fetches the latest version and restarts. Staff simply see a small "reload to update" prompt. Your shop's data is never touched by an update.

Opening the app from home / another location (optional)

To reach this branch over the internet (not just at the shop), there's a free tool called a "Cloudflare Tunnel." See the separate Public URL — Cloudflare Tunnel guide, or ask us to set it up with you.
