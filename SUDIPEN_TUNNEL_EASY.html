BASIC by JMSI · SUDIPEN BRANCH

Open Sudipen from anywhere

The exact tunnel steps for the Sudipen branch. Its online address will be
https://sudipen.basicautomotiveservices.com

Do all of this on the Sudipen mini-PC. First make sure Sudipen's app already works there — open http://localhost:8790/ on that computer and check you get the sign-in page. If not, finish the "Easy Branch Setup" guide first (use SLUG=Sudipen and BRANCH=Sudipen in the settings file).

Two ways. Try the Easy Way first — it can't break anything.

Easy Way — a temporary link to test today (5 minutes, safe)

1 Open the black typing window

Press Windows, type cmd, press Enter.

2 Install the tunnel tool (once)

Type this, then Enterwinget install --id Cloudflare.cloudflared
If it says already installed, that's fine.

3 Turn on the temporary tunnel

Type this, then Entercloudflared tunnel --url http://localhost:8790
You'll see a link like https://random-words.trycloudflare.com. Open it on your phone (even on mobile data) — that's Sudipen! Keep the black window open; the link stops when you close it. Great for a quick test.

Permanent Way — the real address sudipen.basicautomotiveservices.com

Read first. The very first time you use any branded address, someone must connect basicautomotiveservices.com to Cloudflare (a one-time "nameserver" change at the company you bought the website name from). Done wrong, it can take your main website offline for a while. If Pilot or another branch already has a working *.basicautomotiveservices.com address, this one-time part is already done — skip to Step 1 below. If not, do the one-time part with help (see the "Easy Tunnel" guide, Part 2), or ask us.

1 Sign the tool in to Cloudflare (once per mini-PC)

Type this, then Entercloudflared tunnel login

A browser opens. Choose basicautomotiveservices.com and approve. Close the browser when done.

2 Create the Sudipen tunnel

Type this, then Entercloudflared tunnel create sudipen
Write down the long Tunnel ID it shows, and the file path it mentions (it looks like C:\Users\YOURNAME\.cloudflared\THAT-ID.json). You'll need both in Step 3.

3 Make the settings file config.yml

  1. Open Notepad.
  2. Copy the lines below, and replace YOURNAME and THAT-ID with the ones from Step 2:
Copy into Notepad — change YOURNAME and THAT-IDtunnel: THAT-ID credentials-file: C:\Users\YOURNAME\.cloudflared\THAT-ID.json ingress: - hostname: sudipen.basicautomotiveservices.com service: http://localhost:8790 - service: http_status:404
  1. Click File → Save As. Go to the folder C:\Users\YOURNAME\.cloudflared.
  2. Set "Save as type" to All Files (*.*), name it exactly config.yml, and Save.
Make sure the file is named config.yml — not config.yml.txt. (Same trap as before: choose "All Files" first.)

4 Point the address at the tunnel

Type this, then Entercloudflared tunnel route dns sudipen sudipen.basicautomotiveservices.com

5 Make it always-on

Type this, then Entercloudflared service install

Then start it (or just restart the mini-PC):

Type this, then Enternet start cloudflared
Done! Open https://sudipen.basicautomotiveservices.com from any phone or computer, anywhere.

Stay safe (it's on the internet now)

  • Make sure the Sudipen admin password is changed (never admin).
  • Give staff strong passwords.
  • Optional: turn on Cloudflare Access (a free email-code lock before the sign-in page). Ask us if you want it.

If something goes wrong

The trycloudflare link won't openKeep the Step 3 window open, and check http://localhost:8790/ works on the mini-PC.
"cloudflared is not recognized"Close and reopen the black window after installing (Easy Way, Step 2), then try again.
The permanent link doesn't work yetCloudflare may still be setting up. Wait a bit; make sure the service is running (net start cloudflared).
Main website went downStop and contact us — a nameserver setting needs fixing.
